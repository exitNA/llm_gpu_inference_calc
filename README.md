# GPU Analysis

一个基于 Gradio 的单模型推理 GPU 资源估算器，用于交互式评估显存、吞吐和高可用采购规模。
当前交互版以“模型预设 + GPU 规格预设 + 并发量 + 吞吐目标”作为主要输入，适合快速做采购和部署初判。

## 运行

```bash
uv sync
uv run python main.py
```

默认启动 Gradio 页面，监听 `0.0.0.0:7860`。可选参数：

```bash
uv run python main.py --host 127.0.0.1 --port 7860
uv run python main.py --dry-run
```

`--dry-run` 会使用内置示例参数执行一次估算并直接输出文本结果，不启动 Web UI。

## 交互输入

- 模型参数：按模型系列和版本选择预设
- GPU 规格：按 GPU 系列和版本选择预设
- 流量参数：并发量、`target_prefill_tps_total`、`target_decode_tps_total`
- 高可用参数：模式、副本数、故障冗余

## 单位约定

- 显存容量配置使用 `GB`
- 显存展示统一使用 `GB`
- 显存带宽配置使用 `GB/s`
- 模型精度、KV cache 等底层大小使用 `bytes`

## 内置简化假设

- 固定请求画像：50% 轻问答、35% 中等分析、15% 长上下文
- 固定 batch：`1`
- 默认每并发 decode 目标：`20 tok/s`
- 默认每并发 prefill 目标：`500 tok/s`

页面会先按上述固定画像给出默认吞吐目标，你也可以在 UI 中手动覆盖。请求画像里的 token 分布仍主要用于显存估算；若业务吞吐目标发生变化，需要同步调整这两个 TPS 参数。

## 输出结果

- 决策摘要：最终采购卡数、业务基线卡数、主导约束、并发与 token 速度、每日 token 供给上限、平均一次对话耗时、估算总成本
- 约束拆解：显存、吞吐、高可用三类关键指标
- 请求与 KV 明细
- 计算过程明细：展示主要预估值的公式、代入值和结果，便于人工验证
- 原始 JSON 结果，便于继续集成或导出

其中“每日 token 供给上限”按业务基线卡数 `G_biz` 估算，不将 HA 冗余卡重复计入可供给产能；界面会分别展示 `decode` 与 `prefill` 两种口径。

“平均一次对话耗时”按当前业务基线卡数 `G_biz` 的集群能力近似估算，公式为：平均输入 token 的 prefill 时间 + 平均输出 token 的 decode 时间，结果单位为秒。
