# GPU Analysis

一个基于 Gradio 的单模型推理 GPU 资源估算器，用于按文档《`docs/llm_gpu_inference_calc.md`》的方法评估在线推理的显存下界、吞吐下界和时延必要条件。
当前交互版以“模型预设 + GPU 规格预设 + 平均/峰值 QPS + 平均/P95 长度 + TTFT/E2E 目标”作为主要输入，适合做采购前容量规划和能力回推。

## 运行

```bash
uv sync
uv run python main.py
```

默认启动 Gradio 页面，监听 `0.0.0.0:7860`。可选参数：

```bash
uv run python main.py --host 127.0.0.1 --port 7860
uv run python main.py --dry-run
```

`--dry-run` 会使用内置示例参数执行一次估算并直接输出文本结果，不启动 Web UI。

## 交互输入

- 模型参数：按模型系列和版本选择预设
- GPU 规格：按 GPU 系列和版本选择预设，并设置精度、`η_vram`、`η_bw`、`η_cmp`
- 业务目标：平均/峰值 QPS、平均/P95 输入输出长度、TTFT/E2E 目标（单位秒）
- 显存附加系数：`α_w`、`α_r`

## 单位约定

- 显存容量配置使用 `GB`
- 显存展示统一使用 `GB`
- 显存带宽配置使用 `GB/s`
- 模型精度、KV cache 等底层大小使用 `bytes`

## 默认业务画像

- 默认画像沿用仓库原始轻问答/中等分析/长上下文混合分布折算后的平均值与 P95 值
- 默认时延目标以秒级在线问答场景为例
- 峰值在途预算额外乘以 `1.1` 安全系数

## 输出结果

- 决策摘要：业务基线卡数 `G_req`、主导约束、时延风险、估算总成本
- 约束拆解：`G_mem`、`G_pre`、`G_dec`、两条时延必要条件
- 能力回推：总 Prefill / Decode 吞吐、保守/常态可持续 QPS、每日 token 总量、显存口径最大在途请求量
- 业务与 Cache 明细
- 计算过程明细：展示主要预估值的公式、代入值和结果，便于人工验证
- 原始 JSON 结果，便于继续集成或导出

其中“每日 token 供给上限”和“可持续 QPS”都按业务基线卡数 `G_req` 估算。时延相关输出遵循文档定义，只做“单卡能力层面的必要条件检查”，不直接把时延折算为总卡数。
