# GPU Analysis

一个基于 Gradio 的单模型推理 GPU 资源估算器，用于交互式评估显存、吞吐和高可用采购规模。
当前交互版以“模型预设 + GPU 规格预设 + 并发量”作为主要输入，适合快速做采购和部署初判。

## 运行

```bash
uv sync
uv run python main.py
```

默认启动 Gradio 页面，监听 `0.0.0.0:7860`。可选参数：

```bash
uv run python main.py --host 127.0.0.1 --port 7860
uv run python main.py --dry-run
```

`--dry-run` 会使用内置示例参数执行一次估算并直接输出文本结果，不启动 Web UI。

## 交互输入

- 模型参数：按模型系列和版本选择预设
- GPU 规格：按 GPU 系列和版本选择预设
- 流量参数：仅输入并发量
- 高可用参数：模式、副本数、故障冗余

## 内置简化假设

- 固定请求画像：50% 轻问答、35% 中等分析、15% 长上下文
- 固定 batch：`1`
- 固定每并发 decode 目标：`20 tok/s`
- 固定每并发 prefill 目标：`500 tok/s`

也就是说，页面只暴露并发量，但显存和吞吐估算仍会基于以上固定画像自动推导。

## 输出结果

- 决策摘要：最终采购卡数、业务基线卡数、主导约束、估算总成本
- 约束拆解：显存、吞吐、高可用三类关键指标
- 请求与 KV 明细
- 原始 JSON 结果，便于继续集成或导出
