# 交付说明清单（共享）

适用范围：开发、文档整理与改造类 skills 的统一交付输出。

## 必填项

1. 改动内容与文件清单（按层标注：文档/模型或状态/服务或页面/测试）。
2. 改动完整性核对结论（引用 `change-completeness-checklist.md`，列出完成项与证据）。
3. 过期内容清理结果（旧字段/旧公式/旧示例/旧说明的删除情况）。
4. 一致性说明（代码、文档、输出是否已经对齐）。
5. 验证命令与结果（按 `quality-gates.md`）。
6. 剩余风险与后续优化建议。

## 场景补充项

- 公式改动：补充单位、近似前提和样例结果影响。
- 文档重写：补充主文档与次级文档的职责划分。
