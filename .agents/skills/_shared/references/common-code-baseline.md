# 通用代码基线（共享）

适用范围：`server-code-style` 及其他 Python 相关 skill。

## 统一约束

- 函数/方法不超过 80 行；超限时按职责拆分。
- 单文件不超过 500 行；超限时按模块拆分。
- 避免 Magic Number / Magic String；提取为具名常量或枚举。
- 仅允许极少数自解释字面量直接出现（如 `0`、`1`、`-1`）。
- 遵循 DRY；重复逻辑应抽取复用。
- 圈复杂度建议控制在 10 以内。
- 控制复杂度；避免长链路条件与多职责函数。
- 单位、比例和近似前提必须显式表达，避免隐式换算。
- 文档中出现的参数名、默认值和代码保持一致。

## 使用方式

- 各 skill 只保留本领域差异规则。
- 通用基线统一以本文件为准，避免重复维护。
