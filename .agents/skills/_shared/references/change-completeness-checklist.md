# 改动完整性检查清单（共享）

适用范围：文档与 Python 计算逻辑改动的统一核对清单。

## 使用方式（强制）

- 实施前建立“改动面矩阵”，明确代码真源和文档真源。
- 实施后逐项核对本清单；存在未完成项时，不得交付。
- 交付说明必须包含完整性结论与证据（测试结果、检索结果、命令输出摘要）。

## 改动面矩阵（必查）

### 1) 公式与参数定义

- 参数名、默认值、单位、取值范围在代码和文档中保持一致。
- 若修改公式，必须同步检查中间变量名和结果字段名。
- 平均值、P95、吞吐、显存等口径不得混用。

### 2) 代码结构

- 数据类、校验函数、估算函数、打印输出是否同步更新。
- 新增输入是否有默认值和非法输入处理。
- 新增输出是否被 `print_result` 或调用方正确消费。

### 3) 文档与示例

- 主说明文档、正式方案文档和 README 是否仍然各司其职。
- 示例参数、样例输出和正文解释是否与当前实现一致。
- 是否删除了与当前版本冲突的旧段落和重复说明。

### 4) 删除与清理（防僵尸内容）

- 删除旧参数、旧常量、旧标题、旧公式和重复示例。
- 使用 `rg` 检索旧符号或过期路径；结果应为 0 或仅剩注释说明。

### 5) 测试与验证

- 至少完成 `quality-gates.md` 中与改动匹配的最小验证。
- 行为变化时，至少运行一次样例计算或手工核对关键输出。
- 按 `quality-gates.md` 执行门禁并记录结果。
