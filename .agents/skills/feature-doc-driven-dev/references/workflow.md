# 文档驱动工作流

## 1. 变更分类

- `method`: 方法或公式调整。
- `presentation`: 正式方案文档、README 或说明文重写。
- `sync`: 代码与文档同步修正。

## 2. 文档先行顺序

1. 确认主文档：方法说明放 `docs/llm_gpu_inference_calc.md`，仓库入口说明放 `README.md`。
2. 先改主文档的术语、公式、输入输出定义。
3. 再同步次级文档中的参数表、样例和解释。
4. 若行为改变，再修改 `main.py`。
5. 最后删除冲突段落和重复内容。

## 3. 文档门禁

- 一个概念只保留一个主定义位置。
- 文档必须显式写清单位和近似前提。
- 公式、变量名和代码中的字段名要可互相映射。
- 正式版文档允许更完整，但不能和方法说明写出相反结论。

## 4. 实现与验证

- 代码改动遵循 `../server-dev/SKILL.md` 和 `../server-code-style/SKILL.md`。

交付前验证：

- 文档变更范围与代码变更范围一致。
- `docs/*` 中的公式、字段名、默认值与 `main.py` 一致。
- 不存在引用缺失文件或已经废弃的项目结构。
