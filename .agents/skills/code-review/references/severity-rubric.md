# Severity 分级标准

## S0 Blocking

- 会导致数据错误、数据丢失、权限漏洞、核心流程不可用。
- 会破坏既有 source-of-truth 或引入不可接受的一致性风险。
- 必须在合并前修复。

## S1 High

- 高概率触发错误结果、明显回归或线上稳定性问题。
- 有明确用户影响，但存在临时绕行方式。
- 原则上应在合并前修复，至少需要明确缓解方案与负责人。

## S2 Medium

- 中低概率问题、边界条件缺陷、可维护性隐患或观测缺口。
- 建议在当前迭代修复，若延后需记录技术债与计划。

## S3 Low

- 非阻塞优化项（命名、结构、局部重构建议等）。
- 不影响正确性与上线安全。

## 判定规则

- 同时命中多个级别时，取最高级别。
- 没有可复现路径或证据时，不升级 severity；先补证据再结论。
- 风格问题不得上升为 S0/S1，除非已造成可证明的行为风险。
